PSX-INTEWRACTOR

A remote variable editor for interactive artist intervention into running games


PSX-INTEWRACTOR allows you to interactively change the contents of variables of a remotely running program. You can attach this program anytime to a running program, if the communication link does allow remote debugging! For some datatypes special editors exist, like COLOR, MATRIX and VECTOR. The datatypes and symbol names are retrieved from a debugging version of the executable which is currently running on the PSX. Due to technical restrictions it is only possible to edit global static variables, only these are visible to the intewractor.

Application Usage:

Start the intewractor with the program to be debugged as argument. The file format of the program must be an unstripped debugging version in MIPS-ECOFF format (this is what the PSXDEV egcs compiler does produce by default). Typically you would to the following:
 
  1. compile your program with "-g"
  2. keep an unstripped copy (i.e. file.dbg)
  3. strip the original (i.e. file.bfd)
  4. convert it to PSX-EXE (psx-bfd2x -o file.psx file.bfd)
  5. upload and start the PSX-EXE (psx-run -R -S file.psx)
  6. if you're using the pcdrv: file server for initial data uploads, wait until they're finished
  7. interrupt the file server (CTRL-C to psx-run)
  8. start "psx-intewractor file.dbg"
All this activities should be placed in your projects Makefile!
 
  • Revert Button
    Reloads the symbol table (do this after a recompilation..)

  • Resume/Halt/Reset Buttons
    They control the PlayStation, but are not very important.

  • Variable List
    All global static declarations will appear here. Dynamic or local declarations can't be detected.
The datatype buttons below the list view will open a datatype editor for the currently selected variable. You can open as many editors as you like, get rid of them by closing them. You even can open two different editors for the same variable, i.e. to check different things with different values..
