File placement

All files are written alphabetically sorted to the master image, so a file named ZZZZZZZZ.ZZZ;1 would be the last file written to the image. If file placement matters, you can control it with the filenames.

Creating a bootable disc

This is a problematic section. An original PlayStation rejects to boot unlicensed, non-black CD-ROM medias, so you need a developer PlayStation like the black NetYaroze station. I can't help you here if you are using a gray or white end user PlayStation, sorry.

Avoiding laser seek problems

To avoid laser seek problems, you should always include a dummy file at the end of your data track, which is at least three minutes long. Many games have a DUMMY.DAT;1 which is located at the end of the data track, and I encountered seek problems when mastering too small data tracks. To be sure, create a three minute long (~64MB) dummy file at the end of the data track (just name it ZZZZZZZZ.ZZZ;1).

Temporary files

The master process writes an image named image.iso (the name can be changed here) into the output path directory, and the write process creates a TOC description file named image.toc (can also be changed here) in the same location. These files are not deleted, you have to delete them manually!

Saving the settings

Settings are always written to the current directory (the directory where the application was started from) into a file named .psxcdmakerrc. This is a textfile with human readable options, but it is not recommended to edit this file.
