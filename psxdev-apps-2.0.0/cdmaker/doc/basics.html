Basics of a PSX CD-ROM

General

All informations provided here are guessed and not offical, so keep this in mind. I made some test series to find it out (try and error...)

Filesystem

The PSX seems to use a standard ISO 9660 level 1 filesystem (8+3 names plus version suffix ;1). The ISO filesystem implies a maximum directory depth of 8 levels, and it seems that the total number of directories and files on one disc is limited too (it uses only one sector to store this information).
  • up to 45 directories
  • up to 30 files per directory
  • maximum directory depth is 8 levels
  • filenames must conform 8+3 notation plus version suffix

CD-ROM Track types

The first track must always be a data track. Since you may want to embed XA files (which take 2336 bytes/sector) the data track should be of type CD_ROM_XA. See below for more informations on XA files. All following tracks can only be audio tracks, and a typical maximum of 99 tracks in total is supported.

Embedding a product license

The system area of a PSX CD-ROM (the first 16 sectors) contains kind of a product license. I you have such a license file, you can embed it these sectors.

Embedding XA streams

Two file types are supported by PSXDEV CD Maker:
  • Mode 2 Form 1 (Used for all normal files)
    • 8 Bytes Subheader
    • 2048 Bytes Data
    • 4 Bytes EDC
    • 276 Bytes ECC
  • Mode 2 Form 2 (Used for all XA files)
    • 8 Bytes Subheader
    • 2324 Bytes Data
    • 4 Bytes Reserved
You can specify filename filters to select the files to be embedded in XA mode, like *.xa. It is up to you how you create the XA files; I've tested this feature by using some existing XA files (audio files and movies), creating a CD-ROM and replaying them with the help of the CAETLA movie player. There are many details missing yet (like indexing, owner groups and some sync informations) and since I do not have enough informations about this they won't be implemented in the near future.
